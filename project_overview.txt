# 项目文件介绍

## 项目概述
这是一个智能客服助手项目，使用 LangGraph 和 Streamlit 构建，支持用户认证、知识库检索、订单查询和工单创建。数据存储在本地 SQLite 数据库中。

## 文件结构和说明

### 根目录文件

#### docker-compose.yml
Docker Compose 配置文件，用于定义和运行多容器 Docker 应用程序。可能包含应用服务、数据库等。

#### dockerfile
Docker 镜像构建文件，定义如何构建应用的 Docker 镜像。

#### README.md
项目说明文档，包含项目介绍、安装和使用指南。

#### requirements.txt
Python 依赖文件，列出项目所需的所有 Python 包及其版本。
- langchain>=0.2.0: LangChain 框架，用于构建 LLM 应用。
- langgraph>=0.1.0: LangGraph，用于构建有状态的代理工作流。
- langchain-openai>=0.1.0: LangChain OpenAI 集成。
- langchain-community>=0.2.0: LangChain 社区扩展。
- langchain-text-splitters>=0.2.0: 文本分割工具。
- langchain-huggingface>=0.2.0: Hugging Face 集成。
- chromadb>=0.5.0: Chroma 向量数据库。
- sentence-transformers>=2.6.0: 句子嵌入模型。
- streamlit>=1.35.0: Streamlit Web UI 框架。
- python-dotenv>=1.0.0: 环境变量管理。

#### tickets.txt
存储用户创建的工单数据。每行格式：ticket_id,contact,user_issue。

#### .env
环境变量配置文件。
- DEEPSEEK_API_KEY: DeepSeek API 密钥。
- HF_ENDPOINT: Hugging Face 端点，设置为国内镜像 https://hf-mirror.com。

#### orders.json
存储订单数据，JSON 格式，键为手机号，值为订单状态字符串。

#### users.json
存储用户凭据，JSON 格式，键为用户名，值为密码。

### chroma_db/ 目录
向量数据库存储目录。
- chroma.sqlite3: Chroma 数据库文件。
- 30b7ca06-e94b-482e-9fee-f03b70c99986/: Chroma 内部数据目录。

### knowledge_base/ 目录
知识库文档目录。
- 常见问题.txt: 常见问题解答文档。
- 退货政策.txt: 退货政策文档。
- 物流说明.txt: 物流说明文档。

### src/ 目录

#### agent.py
核心代理逻辑文件。
- SqliteSaver 类: 自定义检查点保存器，使用 SQLite 存储 LangGraph 状态。
  - __init__(): 初始化 SQLite 连接。
  - put(config, checkpoint, metadata, new_versions): 保存检查点。
  - get_tuple(config): 获取检查点元组。
  - _serialize_value(value): 序列化值。
  - _deserialize_value(data): 反序列化值。
  - put_writes(config, writes, task_id): 保存写入操作。
  - _load_blobs(config): 加载 blob 数据。
- app: LangGraph 应用实例，包含工具和状态管理。

#### app.py
Streamlit UI 入口文件。
- load_dotenv(): 加载环境变量。
- _normalize_message(msg): 标准化消息为纯文本字符串。
- login(): 用户登录函数，验证用户名密码。
- 主逻辑: 检查登录状态，初始化 thread_id，加载历史消息，显示聊天界面，处理用户输入。

#### build_vectorstore.py
构建向量库脚本。
- 加载 knowledge_base/ 下的文档。
- 分割文本。
- 使用 HuggingFaceEmbeddings 创建嵌入。
- 构建并持久化 Chroma 向量库。

#### tools.py
工具定义文件。
- search_knowledge_base(query): 使用向量库检索知识库，返回相关文档内容。
- query_order(phone): 根据手机号查询订单状态，从 orders.json 读取。
- create_ticket(user_issue, contact): 创建工单，写入 tickets.txt，返回工单号。

## 数据存储
- 聊天历史和状态: 通过 SqliteSaver 存储在本地 chroma.sqlite3 或单独的 SQLite 文件。
- 向量数据: Chroma 数据库。
- 用户数据: users.json。
- 订单数据: orders.json。
- 工单数据: tickets.txt。

## 运行方式
1. 安装依赖: pip install -r requirements.txt
2. 构建向量库: python src/build_vectorstore.py
3. 运行应用: streamlit run src/app.py
